Projet BTS SIO n°1 · E6 — Situation professionnelle

Serveur de messagerie Exchange 2019

Déploiement complet d'un serveur de messagerie d'entreprise sous Windows Server 2025, intégré à l'Active Directory. Gestion des boîtes aux lettres, bases de données dédiées, boîtes partagées et calendrier collaboratif.

  • › VM Windows Server 2025 (16 Go RAM, VLAN 120, IP 192.168.120.251) jointe au domaine
  • › Installation Exchange 2019 CU10 + prérequis (Visual C++, rôle boîtes aux lettres)
  • › Zones DNS (mail / autodiscover), certificat auto-signé déployé par GPO
  • › Bases de données séparées (Direction/IT et Employés), migration de boîtes
  • › Boîtes aux lettres partagées et partage de calendrier inter-utilisateurs
Voir les étapes détaillées

Préparation du serveur

Création de la VM sous Hyper-V (4 vCPU, 16 Go de RAM dynamique, disque 80 Go, carte réseau sur le VLAN 120). Installation de Windows Server 2025 en expérience utilisateur, configuration de l'IP statique 192.168.120.251, renommage de la machine et jonction au domaine ybdt.local.

Installation d'Exchange 2019

Installation des prérequis (redistribuables Visual C++ 2012/2013), montage de l'ISO Exchange et lancement du setup en administrateur. Sélection du rôle « boîte aux lettres », nom d'organisation ybdt, puis exécution des tests de préparation avant installation.

Configuration DNS & certificat

Création des zones DNS mail.ybdt.local et autodiscover.ybdt.local avec leurs enregistrements hôte (A). Génération d'un certificat auto-signé en PowerShell (New-SelfSignedCertificate), affectation au service IIS, export et déploiement automatique sur tous les postes du domaine via une GPO « Certificat Exchange ».

Boîtes aux lettres & bases de données

Création des boîtes via le centre d'administration Exchange (ECP). Mise en place de deux bases de données dédiées — BddVipDB1 (Direction & IT) et BddUserDB1 (Employés) — avec redémarrage du service MSExchangeIS, puis test de migration de boîtes entre bases.

Collaboration

Création de boîtes aux lettres partagées par unité d'organisation (IT, Employés) et activation du partage de calendrier (stratégie de partage + Set-OwaVirtualDirectory). Vérification de l'envoi de mails et du partage de calendrier entre utilisateurs via l'interface web OWA.

Projet BTS SIO n°2 · E6 — Situation professionnelle

Serveur de supervision Centreon

Mise en place d'une solution de supervision réseau sous Debian 12 avec Centreon. Surveillance de l'ensemble du parc (serveurs Windows, switch Cisco, routeur Ubiquiti) via SNMP, avec tableaux de bord et notifications par e-mail.

  • › VM Debian 12 (IP 192.168.120.252), installation Centreon + MariaDB + Apache
  • › Déploiement de l'agent SNMP sur serveurs Windows, switch Cisco et routeur Ubiquiti
  • › GPO pare-feu pour autoriser le port SNMP (UDP 161) sur le domaine
  • › Notifications e-mail via Postfix relayé sur Exchange
  • › Tableaux de bord personnalisés (VLAN 120, routeur + switch)
Voir les étapes détaillées

Installation de Debian & Centreon

Création de la VM Debian 12 (4 vCPU, 8 Go RAM, disque 40 Go, VLAN 120) avec IP 192.168.120.252, passerelle et DNS du domaine. Installation des dépôts (Sury PHP 8.2, MariaDB 10.11, Centreon 25.10), puis de Centreon. Sécurisation de la base MariaDB et démarrage des services (Apache, php-fpm, centengine, gorgoned…).

Configuration web

Installation via l'assistant web (http://192.168.120.252/centreon) : vérification des prérequis, moteur de supervision, compte admin, connexion à la base MariaDB et installation des modules. Réglage de la langue (fr_FR) et du fuseau (Europe/Paris), puis ajout de la licence gratuite Centreon IT-100.

Déploiement SNMP sur le parc

Installation du service SNMP (communauté ybdtspv, accès en lecture seule depuis le superviseur) sur les serveurs Windows (AD, Exchange, Core via PowerShell), le switch Cisco et le routeur Ubiquiti (EdgeOS en CLI). Création d'une GPO « SNMP - SPV » pour ouvrir le port UDP 161 dans le pare-feu du domaine.

Notifications par e-mail

Installation de Postfix sur le serveur Centreon, configuré en relais (relayhost) vers le serveur Exchange (192.168.120.251:25). Activation du connecteur de réception anonyme côté Exchange, puis test d'envoi de mail. Création d'un contact ITsupport recevant les alertes (24x7, host-notify-by-email).

Hôtes & tableaux de bord

Téléchargement des connecteurs de supervision (Windows SNMP, Cisco, Linux, Ubiquiti), création des 6 hôtes avec leurs modèles, et changement massif pour lier le contact ITsupport. Construction de tableaux de bord personnalisés (statuts, métriques CPU/RAM, latence ping) pour le VLAN 120 et l'ensemble routeur + switch.
